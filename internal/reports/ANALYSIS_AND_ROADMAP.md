# Analysis and Roadmap: Vertex Sentinel vs. AI Trading Agent Template

## 🛡️ Strategic Analysis: The Sentinel Layer Perspective
Vertex Sentinel is designed as a **verifiable risk management layer**. While the [cloned template](https://github.com/Stephen-Kimoi/ai-trading-agent-template) provides a robust boilerplate for building an agent, Vertex Sentinel provides the **security guardrails** that make such agents safe for production.

### Key Insight
The template focuses on **Agent Autonomy** (how the agent thinks and acts).
Vertex Sentinel focuses on **Agent Accountability** (how the agent is constrained and verified).

## 🛠️ Missing Essentials (The "Best Thing" Checklist)

### 1. 🧠 Structured Explainability (`/src/explainability`)
The template has a `reasoner.ts` that converts raw trade decisions into human-readable strings.
- **Action:** Create a `src/utils/explainability.ts` to format Genkit's `riskScoreFlow` output into clear "Why did we trade?" summaries for users.

### 2. 📝 Signed Checkpoints (The "Audit Trail")
The template signs every decision (even HOLDs) with EIP-712 and stores them as checkpoints.
- **Action:** Extend our `Authorization` artifact to include a `reasoningHash` and `timestamp`, making the logs tamper-proof on-chain.

### 3. 🚦 Monitoring Dashboard
The template includes a `leaderboard.html`.
- **Action:** Create a simple `docs/LIVE_MONITOR.md` or a basic HTML view that parses our `logs/audit.json` into a readable format for judges.

## 🚀 How to Integrate the Template's Best Parts
1. **Don't Copy Code:** The template uses `ethers`, we use `viem`. Direct copying will break things.
2. **Copy Patterns:** Use the `Checkpoint` pattern. Before sending the `TradeIntent` to the blockchain, save a local signed artifact of *why* the agent decided to trade.
3. **Directory Names:** Use `src/execution/` (done) and `src/logic/` (done). Consider adding `src/monitoring/`.
